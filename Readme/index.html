Description of Project

R-type is a video game developed in C++. It’s a shoot’em up type shooter supporting 1 to 4 players on a network. The project was completed over 7 weeks. The first 4 weeks consisted of developing a prototype gameplay of the game. The last 3 weeks consisted of choosing a track and completing it. Among the tracks we had, we had the Game Engine track, the NetWork Advanced track, the Game Design track. The project being a product, it had to have solid documentation in order to allow people wishing to work on it to better find their way into the code. Solid software engineering practices, such as the use of CMake, autonomous dependency management and cross-platform portability, are required.

Platforms supported by the project

The project requires the game to be cross-platform. Ideally the game would compile and run perfectly on a Linux system, on Window and on MacOs. but currently the systems on which our project runs are Linux and Windows.

Installation

$ ./build.sh

Build

For a smooth running of the gameplay of the game and in order to avoid having technical problems, certain prerequisites are mandatory and essential.

  • You need to install Visual Studio.
  • You need to install the Visual Studio build tools.
  • You need to install C++ and Cmake dependencies in Visual Studio.
  • You need to install Conan which is a C++ package manager.

Compilation using cmake

  • First of all, you need to create a directory named build using 'mkdir dirname'
  • Second of all, you need to go to the directory using 'cd build'

Reqirements

In order to ensure a favorable evaluation for the defense and to maximize the allocation of the six credits to the project, it is imperative to meet certain prerequisites. Among these conditions, we find:

  • The development of a product, as opposed to the execution of a project.
  • Compilation management with CMake.
  • Management of third-party dependencies.
  • Cross-Platform Management: The project must be able to compile on both Linux and Windows.
  • Version control flow management, which includes having proper Git practices and automated Continuous Integration (CI).
  • Dependency packagings.

Usage

Once the compilation is complete, you launch the server binary and the client binary. You will be presented with a client-level interface. On this interface you will have to enter your nickname, your IP address, the port, the room that you would like to integrate. In addition, information such as choosing your space shuttle is also available on the interface.

Contributing

If you wish to contribute to the project, you must already contact the project contributors. You can write to them via their emails. Then you need to read the documentation and information about the project. Once these two elements are established, you can start coding.
